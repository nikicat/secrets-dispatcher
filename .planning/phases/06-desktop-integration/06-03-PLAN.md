Create `internal/agent/agent.go` — a new package that runs the Secret Service proxy as a desktop user agent.

**Architecture overview:** The existing `internal/proxy/` package already implements the full Secret Service API proxy (Service, Collection, Item handlers). It was designed to connect to a "remote" D-Bus and a "local" D-Bus. In v1.0, "remote" was the forwarded socket from another machine and "local" was the desktop session bus where the real Secret Service lives.

In Phase 6, the topology inverts:
- **Frontend (desktop session bus):** The agent registers `org.freedesktop.secrets` here — this is where Chrome/Firefox/libsecret call.
- **Backend (companion session bus):** The real Secret Service (gopass-secret-service) runs here.

The agent needs to:
1. Connect to the desktop user's own session bus (frontend): `dbus.ConnectSessionBus()`
2. Connect to the companion's session bus (backend): `dbus.Connect("unix:path=/run/user/<companion-uid>/bus")`
3. Register Secret Service handlers on the frontend bus that forward to the backend bus
4. Block until context cancellation

**Config struct:**
```go
type Config struct {
    // DesktopUsername is the current user's login name.
    DesktopUsername string
    // CompanionBusAddress overrides the companion bus address (test seam).
    // Empty = auto-compute from companion UID.
    CompanionBusAddress string
    // LogLevel for the proxy logger.
    LogLevel slog.Level
}
```

**func Run(ctx context.Context, cfg Config) error:**
1. Look up companion bus address (same `companionBusAddress` pattern as gpgsign):
   - `user.Lookup("secrets-" + cfg.DesktopUsername)` to get companion UID
   - `fmt.Sprintf("unix:path=/run/user/%s/bus", u.Uid)` for the address
   - Or use `cfg.CompanionBusAddress` if non-empty (test seam)
2. Connect to companion session bus: `companionConn, err := dbus.Connect(addr)`
   - On error: return `fmt.Errorf("connect to companion session bus: %w — is the companion running?", err)`
   - This satisfies AGENT-05: immediate error, no hang
3. Connect to desktop session bus: `desktopConn, err := dbus.ConnectSessionBus()`
4. **Register Secret Service on desktop bus:**
   - The existing proxy package's handler structs (Service, CollectionHandler, ItemHandler) handle method calls by forwarding to a backend connection. The agent creates these handlers pointing at the companion connection as backend.
   - **CRITICAL ANALYSIS NEEDED:** Read `internal/proxy/proxy.go`, `service.go`, `collection.go`, `item.go` to understand how `localConn` and `remoteConn` are used. In v1.0:
     - `remoteConn` = remote D-Bus from SSH-forwarded socket (where proxy registers `org.freedesktop.secrets`)
     - `localConn` = local session bus (where the real Secret Service lives)
   - In Phase 6 agent:
     - `desktopConn` = desktop session bus (where agent registers `org.freedesktop.secrets`) — was `remoteConn`
     - `companionConn` = companion session bus (where gopass-secret-service runs) — was `localConn`
   - So the mapping is: the existing Proxy's `remoteConn`→`desktopConn`, `localConn`→`companionConn`
   - **Approach:** Use `proxy.New()` and `proxy.Config` but adapt the connection setup. Since `proxy.Config.RemoteSocketPath` expects a socket path and Connect() dials both connections, the agent may need to either:
     (a) Create a Proxy manually and inject the two connections, or
     (b) Add a new constructor/config option to Proxy for pre-connected connections
   - **Preferred (simpler):** Option (b) — add `FrontendConn` and `BackendConn` fields to `proxy.Config`. When both are set, `Connect()` skips its own dialing and uses the provided connections. This is a minimal change to proxy.go.
   - If modifying proxy.Config is too invasive, option (a): create the proxy struct fields directly in the agent package (duplicate the handler setup). But this duplicates ~100 lines of handler registration. Prefer (b).

5. Request name `org.freedesktop.secrets` on desktop bus: `desktopConn.RequestName("org.freedesktop.secrets", dbus.NameFlagDoNotQueue)` — if it fails, another Secret Service is already running, return error.
6. Block on `<-ctx.Done()`, clean up connections.

**Approval bypass:** The existing proxy uses an `approval.Manager` to gate secret access. In the agent, approval is handled by the companion daemon's VT TUI (the D-Bus call to companion blocks until approved). The agent does NOT need its own approval manager — it's a transparent forwarder. Pass `nil` for approval in the proxy config, or use a pass-through mode.

**IMPORTANT NOTE ON PROXY REUSE:** After analyzing the proxy package, the executor should determine the cleanest integration path. The proxy package was built for a different topology. If reusing it cleanly is not feasible in one task (>60 min estimated), simplify: create a minimal D-Bus forwarder in the agent package that handles only the practical subset of Secret Service methods (OpenSession, SearchItems, GetSecrets, CreateItem, Unlock) by manually registering handler methods on the desktop bus that call the companion bus. This is ~200 lines and avoids modifying proxy.go. The CONTEXT.md says "Existing pipeline architecture handles both proxy and VT-approval cases via configuration" but if the proxy package needs significant refactoring to support the new topology, the minimal forwarder is the pragmatic choice.

```go
func runAgent(args []string) {
    fs := flag.NewFlagSet("agent", flag.ExitOnError)
    logLevel := fs.String("log-level", "info", "Log level: debug, info, warn, error")
    companionBus := fs.String("companion-bus", "", "Companion bus address (default: auto-detect from companion UID)")
    fs.Parse(args)

    level := parseLogLevel(*logLevel)
    // Set up slog (same pattern as runDaemon)
    ...

    // Get current user
    currentUser, err := user.Current()
    if err != nil {
        slog.Error("cannot determine current user", "error", err)
        os.Exit(1)
    }

    ctx, stop := signal.NotifyContext(context.Background(), syscall.SIGTERM, syscall.SIGINT)
    defer stop()

    cfg := agent.Config{
        DesktopUsername:      currentUser.Username,
        CompanionBusAddress: *companionBus,
        LogLevel:            level,
    }

    if err := agent.Run(ctx, cfg); err != nil {
        slog.Error("agent error", "error", err)
        os.Exit(1)
    }
}
```

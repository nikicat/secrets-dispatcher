**run.go ã Rewrite Run() to use CompanionDBusClient:**

Replace steps 5-11 (auth token, socket path, DaemonClient, WebSocket, POST, WaitForResolution) with:

1. Look up the current user: `currentUser, err := user.Current()` ã use `currentUser.Username` for `NewCompanionDBusClient()`
2. Create client: `client, err := NewCompanionDBusClient(currentUser.Username)`
   - On error: `fmt.Fprintf(os.Stderr, "secrets-dispatcher: companion not running: %v\n", err)` ã return 2
3. `defer client.Close()`
4. Print status: `fmt.Fprintf(os.Stderr, "secrets-dispatcher: Waiting for approval on VT8...\n")`
5. Create context with signal handling (keep existing SIGINT/SIGTERM trap pattern, but simpler: just cancel the context)
6. Call `client.RequestSign(ctx, repoName, message, author, committer, keyID, changedFiles, string(commitBytes))`
7. Handle result:
   - `errors.Is(err, ErrDenied)` ã stderr "signing request denied by user", return 1
   - `errors.Is(err, ErrTimeout)` ã stderr "signing request timed out", return 1
   - `errors.Is(err, ErrGPGFailed)` ã write gpgStatus to stderr, extract exit code from error, return it
   - `errors.Is(err, ErrNotReady)` ã stderr "companion daemon not ready", return 2
   - Other error ã stderr error, return 2
   - Success ã write signature to stdout, gpgStatus to stderr, return 0

The GPG thin client per CONTEXT.md: "Does not mimic gpg status-fd format ã simple exit code + human-readable error." So on success write the raw signature bytes to stdout and status bytes to stderr (matching how git expects gpg.program to behave: signature on stdout, status on fd 2).

**Keep unchanged:** Steps 1-4 (key ID parsing, stdin reading, commit parsing, git context collection). These don't depend on transport.

**Remove:** Steps 5-6 (loadAuthToken, unixSocketPath) ã no longer needed with D-Bus transport. Also remove the `loadAuthToken()` and `unixSocketPath()` functions. They can stay as dead code for now if other code paths use them, but check: `loadAuthToken` is only used in `Run()`, `unixSocketPath` is only used in `Run()`. Both can be removed.

**daemon.go ã Deprecate/remove HTTP/WS client:**

The `DaemonClient`, `NewDaemonClient`, `DialWebSocket`, `PostSigningRequest`, `CancelSigningRequest`, `WaitForResolution`, and `wsMsg` types are now unused (the HTTP/WebSocket transport is replaced by D-Bus).

Check if anything else in the codebase imports these. If only `run.go` uses them, remove the entire file `daemon.go`. If there are other consumers, leave it but add a deprecation comment.

Also remove the `loadAuthToken()` and `unixSocketPath()` helper functions from `run.go` if they have no other callers.

**Signal handling simplification:**

The existing signal handler creates a WebSocket cancel context and calls `CancelSigningRequest` on interrupt. With D-Bus:
- Use `signal.NotifyContext(ctx, syscall.SIGINT, syscall.SIGTERM)` for clean cancellation
- The D-Bus call will be cancelled when ctx is done, causing it to return an error
- No need for explicit request cancellation ã the D-Bus method call simply returns early

Create `internal/daemon/logind.go` with the logind session watcher.

**Key design:** The companion daemon connects to the **system bus** specifically for logind signal subscription (even though user-to-companion IPC is on session bus per Phase 6). This is correct because `org.freedesktop.login1.Manager` is a system service — it only exists on the system bus.

**Types and functions:**

```go
// sessionChecker abstracts logind session queries for testability.
type sessionChecker interface {
    // HasActiveSessions returns true if the given username has active logind sessions.
    HasActiveSessions(username string) (bool, error)
}
```

**`WatchDesktopSessions(ctx context.Context, desktopUser string, onAllSessionsGone func())`:**

1. Connect to the **system bus**: `conn, err := dbus.ConnectSystemBus()` (this is separate from the daemon's session bus connection)
   - If connection fails (e.g., running in container without system bus), log warning and return without watching — the daemon simply won't auto-stop. This is a graceful degradation.
2. Add match signal for `org.freedesktop.login1.Manager.SessionRemoved`:
   ```go
   conn.AddMatchSignal(
       dbus.WithMatchInterface("org.freedesktop.login1.Manager"),
       dbus.WithMatchMember("SessionRemoved"),
   )
   ```
3. Create signal channel and subscribe: `conn.Signal(ch)`
4. In a goroutine, loop on signals:
   - For each `SessionRemoved` signal, call `desktopUserHasSessions(conn, desktopUser)`
   - If no sessions remain, call `onAllSessionsGone()` and return
   - On `ctx.Done()`, clean up and return
5. Return immediately (the goroutine runs in background)

**`desktopUserHasSessions(conn *dbus.Conn, username string) (bool, error)`:**

Call `org.freedesktop.login1.Manager.ListSessions`:
```go
obj := conn.Object("org.freedesktop.login1", "/org/freedesktop/login1")
call := obj.Call("org.freedesktop.login1.Manager.ListSessions", 0)
```

The return type is `[]struct{ ID string; UID uint32; Name string; Seat string; Path dbus.ObjectPath }`. Iterate and check if any session has `Name == username`. Return true if at least one found.

**NOTE:** The actual ListSessions return format from logind uses a D-Bus struct `(susso)`. In godbus, this maps to `[][]interface{}` or a struct. Verify the exact deserialization. A safe approach: use `call.Store(&sessions)` with `sessions` as `[][]interface{}` and access fields by index:
- Index 0: session ID (string)
- Index 1: UID (uint32)
- Index 2: user name (string)
- Index 3: seat (string)
- Index 4: session object path (dbus.ObjectPath)

**daemon.go — Add DesktopUser to Config and wire logind watcher:**

1. Add `DesktopUser string` field to `Config` struct with doc comment: "DesktopUser is the login name of the desktop user. Used by the logind watcher to detect when all desktop sessions have ended (COMP-04)."

2. In `Run()`, after the TUI/headless setup and before the final `select`/block:
   - If `cfg.DesktopUser != ""` (production — tests may not set this):
     ```go
     // COMP-04: Watch for desktop user logout to trigger daemon self-stop.
     logindCtx, logindCancel := context.WithCancel(ctx)
     defer logindCancel()
     logindDone := make(chan struct{})
     WatchDesktopSessions(logindCtx, cfg.DesktopUser, func() {
         slog.Info("desktop user has no remaining sessions, shutting down",
             "desktop_user", cfg.DesktopUser)
         close(logindDone)
     })
     ```
   - Add `logindDone` to the `select` in both TUI and headless paths:
     - TUI path: add `case <-logindDone: program.Quit(); <-tuiDone`
     - Headless path: add `case <-logindDone:` alongside `<-ctx.Done()`
